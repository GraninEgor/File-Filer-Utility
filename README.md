# File Filer Utility

Это консольная утилита на Java, которая может обрабатываь несколько текстовых файлов, классифицирует каждую строку как целое число, число с плавающей запятой или строку, а затем записывает отсортированные строки в отдельные выходные файлы. Утилита также предоставляет статистику по обработанным данным.

## Возможности

*   Сортировка содержимого входных файлов на три типа данных: целые числа, числа с плавающей запятой и строки.
*   Запись классифицированных данных в отдельные выходные файлы (`integers.txt`, `floats.txt`, `strings.txt`).
*   Обработка нескольких входных файлов за один запуск.
*   Предоставление параметров командной строки для настройки расположения выходных файлов и префиксов имен.
*   Поддержка как перезаписи, так и добавления данных в выходные файлы.
*   Отображение краткой и, при необходимости, детальной статистики по обработанным данным.

## Начало работы

### Требования

*   Java 17 или выше
*   Apache Maven

### Сборка проекта

1.  Клонируйте репозиторий на локальный компьютер:
    ```sh
    git clone https://github.com/GraninEgor/File-Filer-Utility.git
    cd File-Filer-Utility
    ```
2.  Постройте исполняемый JAR-файл с помощью Maven:
    ```sh
    mvn clean package
    ```
    Это создаст файл `File-Filter-Utility-1.0-SNAPSHOT.jar` в директории `target/`.

## Использование

Запустите утилиту из командной строки, указав параметры и список входных файлов.

### Синтаксис

```sh
java -jar target/File-Filter-Utility-1.0-SNAPSHOT.jar [опции] <входной_файл1> [входной_файл2] ...
```

### Опции

| Опция | Аргумент  | Описание                                                                                          |
| :----- | :-------- | :------------------------------------------------------------------------------------------------- |
| `-o`   | `<путь>`  | Указывает относительный путь к выходной директории. По умолчанию используется `out/`.            |
| `-p`   | `<префикс>`| Добавляет префикс к именам выходных файлов (например, `prefix-integers.txt`).                     |
| `-a`   |           | Включает режим добавления. Данные будут добавляться в существующие файлы, а не перезаписывать их.|
| `-f`   |           | Включает режим полной статистики, который выводит детальную информацию (min, max, sum, avg) для каждого типа данных.|

### Примеры

**Простейшее использование**

Обработка файлов `in1.txt` и `in2.txt` с записью в директорию по умолчанию `out/`.

```sh
java -jar target/File-Filter-Utility-1.0-SNAPSHOT.jar in1.txt in2.txt
```

**Продвинутое использование**

Обработка файлов `in1.txt` и `in2.txt`, добавление префикса `result-` к выходным файлам, размещение их в директории `my_results/`, добавление данных в существующие файлы и вывод полной статистики.

```sh
java -jar target/File-Filter-Utility-1.0-SNAPSHOT.jar -p result- -o my_results -a -f in1.txt in2.txt
```

## Выходные данные

### Файловый вывод

Утилита создаёт до трёх файлов в зависимости от содержимого входных файлов:

*   `integers.txt`: содержит все строки, определённые как целые числа.
*   `floats.txt`: содержит все строки, определённые как числа с плавающей запятой.
*   `strings.txt`: содержит все строки, которые не являются допустимыми целыми числами или числами с плавающей запятой.

По умолчанию файлы создаются в директории `out` в текущей рабочей папке.
